4. Paketverwaltung

Zurück zum Start

Die Paketverwaltung ist ein zentraler Bestandteil der Linux-Administration. Sie ermöglicht es, Software zu installieren, zu aktualisieren und zu entfernen. Jede Distribution hat ihre eigene Paketverwaltung, aber viele von ihnen verwenden ähnliche Konzepte, wie Repositories (Paketquellen), die Softwarepakete enthalten.

Paketmanagement mit apt (Debian, Ubuntu)

Die apt-Paketverwaltung (Advanced Package Tool) wird in Debian und Ubuntu sowie deren Derivaten verwendet. Mit apt können Pakete aus offiziellen Repositories heruntergeladen, installiert, aktualisiert und entfernt werden.

  • apt update: Aktualisiert die lokale Paketdatenbank, um sicherzustellen, dass die neuesten Versionen von Paketen verfügbar sind.
    Beispiel: sudo apt update - Aktualisiert die Paketdatenbank.
  • apt upgrade: Aktualisiert alle installierten Pakete auf die neuesten verfügbaren Versionen.
    Beispiel: sudo apt upgrade - Aktualisiert alle Pakete.
  • apt install: Installiert ein neues Paket aus den Repositories.
    Beispiel: sudo apt install firefox - Installiert den Webbrowser Firefox.
  • apt remove: Entfernt ein installiertes Paket.
    Beispiel: sudo apt remove firefox - Entfernt Firefox vom System.
  • apt autoremove: Entfernt automatisch nicht mehr benötigte Pakete, die von anderen Programmen nicht mehr verwendet werden.
    Beispiel: sudo apt autoremove - Entfernt nicht mehr benötigte Pakete.

Paketmanagement mit dnf (Fedora)

Die dnf-Paketverwaltung ist die Standardpaketverwaltung in Fedora und anderen Red Hat-basierten Distributionen. Sie bietet eine einfachere und schnellere Möglichkeit, Pakete zu verwalten.

  • dnf check-update: Überprüft, ob Updates für installierte Pakete verfügbar sind.
    Beispiel: sudo dnf check-update - Überprüft auf verfügbare Updates.
  • dnf install: Installiert ein neues Paket aus den Repositories.
    Beispiel: sudo dnf install firefox - Installiert Firefox.
  • dnf update: Aktualisiert alle installierten Pakete auf die neueste Version.
    Beispiel: sudo dnf update - Aktualisiert alle Pakete.
  • dnf remove: Entfernt ein installiertes Paket.
    Beispiel: sudo dnf remove firefox - Entfernt Firefox.

Paketmanagement mit yum (CentOS, RHEL)

Das yum (Yellowdog Updater, Modified)-Tool wird in älteren Versionen von Red Hat Enterprise Linux (RHEL) und CentOS verwendet. In neueren Versionen wurde es durch dnf ersetzt, aber viele Systeme verwenden immer noch yum für Paketverwaltung.

  • yum update: Aktualisiert alle installierten Pakete auf die neueste Version.
    Beispiel: sudo yum update - Aktualisiert alle Pakete.
  • yum install: Installiert ein neues Paket aus den Repositories.
    Beispiel: sudo yum install firefox - Installiert Firefox.
  • yum remove: Entfernt ein installiertes Paket.
    Beispiel: sudo yum remove firefox - Entfernt Firefox.

Installation von Software-Paketen

Die Installation von Software erfolgt in der Regel aus den offiziellen Repositories, die von der Distribution bereitgestellt werden. Dies stellt sicher, dass die Software kompatibel mit dem System ist und regelmässig Updates erhält.

Zusätzlich zu den standardmässigen Paketquellen bieten einige Distributionen die Möglichkeit, zusätzliche Software-Quellen zu konfigurieren, sogenannte Personal Package Archives (PPA) für Ubuntu und Debian oder zusätzliche Repositories für andere Distributionen.

Repositorys und PPA (Personal Package Archives)

Repositories sind Sammlungen von Software-Paketen, die über das Internet zugänglich sind. Sie ermöglichen es dem System, Software zu installieren, die durch die Distribution gepflegt wird. Benutzer können auch PPA hinzufügen, um Software zu installieren, die nicht in den offiziellen Repositories enthalten ist.

Beispiel: Hinzufügen eines PPAs in Ubuntu

  • sudo add-apt-repository ppa:example/ppa: Fügt ein neues PPA zu den Quellen von Ubuntu hinzu.
    Beispiel: sudo add-apt-repository ppa:some/ppa - Fügt das PPA `some/ppa` hinzu.
  • sudo apt update: Aktualisiert die Paketdatenbank nach dem Hinzufügen eines neuen PPAs.

Softwarequellen konfigurieren

Die Konfiguration von Softwarequellen ist oft erforderlich, wenn du neue Repositories hinzufügst oder standardmässig auf eine andere Quelle umschalten möchtest. Diese Konfiguration erfolgt normalerweise durch Bearbeiten von Dateien wie /etc/apt/sources.list (für Debian-basierte Systeme) oder durch Verwendung von GUI-Tools wie dem Software & Update-Manager in Ubuntu.

Du kannst auch die Konfiguration von Repositories in anderen Distributionen wie Fedora über dnf oder CentOS über yum anpassen, um neue Quellen zu aktivieren.
