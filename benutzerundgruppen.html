3. Benutzer und Gruppenverwaltung

Erstellen und Verwalten von Benutzern

In Linux-Systemen sind Benutzer die grundlegenden Entitäten, die auf das System zugreifen und mit ihm interagieren. Jede Datei und jedes Verzeichnis im Linux-Dateisystem gehört einem bestimmten Benutzer und einer bestimmten Gruppe. Das Erstellen und Verwalten von Benutzern ist ein wichtiger Aspekt der Systemadministration.

  • useradd: Dieser Befehl wird verwendet, um einen neuen Benutzer zu erstellen.
    Beispiel: sudo useradd max - Erstellt einen neuen Benutzer mit dem Namen `max`.
  • usermod: Mit diesem Befehl können Benutzerkonten geändert werden, z. B. um den Benutzernamen zu ändern oder den Benutzer in eine neue Gruppe zu verschieben.
    Beispiel: sudo usermod -aG sudo max - Fügt den Benutzer `max` zur Gruppe `sudo` hinzu, um Administratorrechte zu erhalten.
  • userdel: Mit diesem Befehl wird ein Benutzer gelöscht.
    Beispiel: sudo userdel max - Löscht den Benutzer `max` vom System.

Erstellen und Verwalten von Gruppen

Gruppen in Linux ermöglichen es, mehreren Benutzern gleichzeitig dieselben Berechtigungen zu erteilen. Benutzer können Mitglied mehrerer Gruppen sein, und die Verwaltung von Gruppen erleichtert die Rechtevergabe und -kontrolle.

  • groupadd: Dieser Befehl wird verwendet, um eine neue Gruppe zu erstellen.
    Beispiel: sudo groupadd admin - Erstellt die Gruppe `admin`.
  • groupdel: Löscht eine Gruppe aus dem System.
    Beispiel: sudo groupdel admin - Löscht die Gruppe `admin`.
  • gpasswd: Mit diesem Befehl kann ein Gruppenadministrator gesetzt oder das Passwort einer Gruppe geändert werden.
    Beispiel: sudo gpasswd -A max admin - Macht den Benutzer `max` zum Administrator der Gruppe `admin`.

Zugriffsrechte und ACLs

In Linux werden Zugriffsrechte über eine Kombination von Benutzer-, Gruppen- und anderen Berechtigungen gesteuert. Diese Berechtigungen legen fest, wer auf Dateien zugreifen darf und welche Operationen (Lesen, Schreiben, Ausführen) erlaubt sind.

  • Dateiberechtigungen: Jede Datei hat drei Arten von Berechtigungen:
    • Lesen (`r`): Erlaubt das Anzeigen des Inhalts der Datei.
    • Schreiben (`w`): Erlaubt das Bearbeiten der Datei.
    • Ausführen (`x`): Erlaubt das Ausführen der Datei als Programm.
  • chmod (Change Mode): Mit diesem Befehl können die Berechtigungen einer Datei geändert werden.
    Beispiel: chmod u+x datei.sh - Fügt der Datei `datei.sh` das Ausführungsrecht für den Besitzer hinzu.
  • chown (Change Owner): Ändert den Eigentümer und/oder die Gruppe einer Datei.
    Beispiel: sudo chown max:admin datei.txt - Ändert den Besitzer der Datei `datei.txt` auf `max` und die Gruppe auf `admin`.
  • setfacl (Set Access Control List): Mit diesem Befehl können ACLs gesetzt werden.
    Beispiel: setfacl -m u:max:rw datei.txt - Gibt dem Benutzer `max` Lese- und Schreibrechte auf die Datei `datei.txt`.
  • getfacl (Get Access Control List): Zeigt die ACLs einer Datei an.
    Beispiel: getfacl datei.txt - Gibt die ACLs für die Datei `datei.txt` aus.

sudo und root-Rechte

In Linux-Systemen ist der root-Benutzer der Superuser und hat uneingeschränkten Zugriff auf alle Systemressourcen. Der Zugriff auf root-Rechte wird normalerweise durch den Befehl sudo gewährt, um sicherzustellen, dass Administratorrechte nur für autorisierte Benutzer zugänglich sind.

  • sudo: Mit diesem Befehl kann ein Benutzer Programme mit Administratorrechten ausführen.
    Beispiel: sudo apt-get update - Führt den Befehl `apt-get update` mit Administratorrechten aus.
  • visudo: Dieser Befehl öffnet die sudoers-Datei, die die Rechte von Benutzern und Gruppen verwaltet, um sudo-Zugriff zu gewähren.
    Beispiel: sudo visudo - Öffnet die Datei zur Konfiguration der `sudo`-Berechtigungen.

Passwortverwaltung

In Linux können Passwörter für Benutzer über verschiedene Befehle verwaltet werden:

  • passwd: Mit diesem Befehl können Passwörter für Benutzer geändert werden.
    Beispiel: sudo passwd max - Ändert das Passwort für den Benutzer `max`.
  • chage: Dieser Befehl wird verwendet, um die Passwortablaufzeit und die Mindestdauer für das Passwort eines Benutzers festzulegen.
    Beispiel: sudo chage -l max - Zeigt die Passwortinformationen für den Benutzer `max` an.

Weitere Tipps zur Benutzerverwaltung

  • Versteckte Benutzer: Einige Linux-Programme benötigen spezielle Benutzerkonten, die keine Login-Rechte haben und in der Regel nicht im System erscheinen. Diese Benutzer beginnen häufig mit einem Unterstrich (z.B. _www für den Webserver).
  • Benutzergruppen: Benutzergruppen sind eine effektive Möglichkeit, Benutzern mit ähnlichen Aufgaben oder Rechten gemeinsam Berechtigungen zuzuweisen.
