10. Virtualisierung

Zurück zum Start

Virtualisierung ist eine Technologie, die es ermöglicht, mehrere virtuelle Maschinen (VMs) auf einem einzigen physischen Host-System zu betreiben. Diese Technologie wird häufig in Serverumgebungen eingesetzt, um die Ressourcennutzung zu optimieren, die Isolierung zu erhöhen und die Verwaltung von IT-Infrastrukturen zu vereinfachen. In diesem Abschnitt wird die Virtualisierung unter Linux behandelt, einschliesslich der Verwendung von KVM, VirtualBox, VMware und Docker.

Einführung in Virtualisierung

Virtualisierung ermöglicht es, mehrere virtuelle Instanzen eines Betriebssystems auf einer einzigen physischen Maschine auszuführen. Dies hat viele Vorteile, darunter die Reduzierung von Hardwarekosten und die Verbesserung der Flexibilität. Es gibt verschiedene Arten von Virtualisierung, die jeweils unterschiedliche Anforderungen und Anwendungsfälle abdecken:

  • Vollvirtualisierung: Bei der Vollvirtualisierung wird die gesamte Hardware des Hosts simuliert. Jede virtuelle Maschine (VM) läuft unabhängig und hat ihre eigene virtuelle Hardware. Beispiele hierfür sind KVM, VMware und VirtualBox.
  • Paravirtualisierung: Hierbei interagieren die VMs direkt mit dem Hypervisor und nutzen dessen Ressourcen effizienter. Paravirtualisierung erfordert, dass die Gastbetriebssysteme speziell für diese Technik angepasst werden. Ein Beispiel hierfür ist Xen.
  • Containerisierung: Container sind leichtgewichtige VMs, die keine eigene vollständige virtuelle Hardware benötigen. Sie teilen sich denselben Kernel, bieten jedoch eine isolierte Umgebung für Anwendungen. Docker und Podman sind bekannte Beispiele für Container-Technologien.

Virtualisierung mit KVM

KVM (Kernel-based Virtual Machine) ist die am häufigsten verwendete Virtualisierungstechnologie auf Linux. Es wird direkt in den Linux-Kernel integriert und bietet eine vollständige Virtualisierung. KVM ermöglicht es, mehrere virtuelle Maschinen auf einem Host zu betreiben, wobei jede VM ein vollständiges, eigenständiges System ist.

  • Installation von KVM: Um KVM auf einem Debian-basierten System wie Ubuntu zu installieren, verwende den folgenden Befehl: 
    sudo apt install qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils virt-manager
  • Überprüfen der KVM-Installation: Um sicherzustellen, dass KVM auf deinem System läuft, kannst du den folgenden Befehl verwenden: 
    sudo kvm-ok
    Dies zeigt an, ob dein Prozessor Virtualisierung unterstützt und ob KVM korrekt installiert ist.
  • Virt-Manager: Virt-Manager ist eine grafische Benutzeroberfläche, die zur Verwaltung von KVM-basierten VMs verwendet werden kann. Nach der Installation kannst du Virt-Manager starten, um VMs zu erstellen und zu verwalten: 
    virt-manager

Virtualisierung mit VirtualBox

VirtualBox ist eine beliebte Virtualisierungssoftware, die sowohl auf Windows als auch auf Linux läuft. Sie ermöglicht es, mehrere virtuelle Maschinen auf einem einzigen physischen Host zu betreiben. VirtualBox ist einfach zu installieren und zu verwenden, und es bietet eine benutzerfreundliche grafische Oberfläche.

  • Installation von VirtualBox: VirtualBox kann auf einem Debian-basierten System mit folgendem Befehl installiert werden: 
    sudo apt install virtualbox
  • Erstellen einer neuen VM in VirtualBox: Nach der Installation von VirtualBox kannst du eine neue VM über die grafische Oberfläche erstellen, indem du auf "Neu" klickst und die erforderlichen Parameter wie Betriebssystemtyp, Speicher und Festplatte festlegst.

Virtualisierung mit VMware

VMware ist eine der führenden Virtualisierungsplattformen und wird hauptsächlich in Unternehmensumgebungen eingesetzt. VMware bietet Lösungen für die Server- und Desktop-Virtualisierung. Es gibt eine kostenlose Version namens VMware Workstation Player, die für private und nicht-kommerzielle Nutzung geeignet ist.

  • Installation von VMware Workstation Player: VMware Workstation Player kann von der VMware-Website heruntergeladen werden. Nach dem Herunterladen des Installationspakets kannst du es mit den folgenden Befehlen installieren: 
    sudo apt install build-essential
    Lade dann das Installationspaket von der VMware-Website herunter und installiere es manuell.
  • Erstellen einer neuen VM in VMware: Ähnlich wie bei VirtualBox kannst du eine neue VM in VMware erstellen, indem du die entsprechende ISO-Datei auswählst und die Installation über die grafische Oberfläche durchführst.

Containerisierung mit Docker und Podman

Containerisierung ist eine leichtgewichtige Form der Virtualisierung, bei der Anwendungen in isolierten Containern ausgeführt werden. Docker und Podman sind zwei der bekanntesten Tools für die Containerisierung. Container teilen sich denselben Kernel, was sie ressourcenschonender macht als virtuelle Maschinen.

  • Docker: Docker ist eine Plattform, die es ermöglicht, Anwendungen in Containern zu entwickeln, bereitzustellen und auszuführen. Docker stellt sicher, dass eine Anwendung und ihre Abhängigkeiten immer in der gleichen Umgebung ausgeführt werden.
  • Installation von Docker: Um Docker auf einem Debian-basierten System zu installieren, führe die folgenden Befehle aus: 
    sudo apt update
    sudo apt install docker.io
  • Docker-Container starten: Um einen Docker-Container zu starten, verwende den folgenden Befehl: 
    sudo docker run -it ubuntu bash
    Dieser Befehl startet einen Container mit der Ubuntu-Distribution und öffnet eine interaktive Bash-Shell.
  • Podman: Podman ist ein weiteres Containerization-Tool, das als Open-Source-Alternative zu Docker dient. Es bietet die gleiche Funktionalität, jedoch ohne die Notwendigkeit eines zentralen Daemons, was es als benutzerfreundlicher und sicherer macht.
  • Installation von Podman: Um Podman auf einem Debian-basierten System zu installieren, führe diesen Befehl aus: 
    sudo apt install podman

Virtualisierungs-Tools und -Befehle

Abhängig von der verwendeten Virtualisierungslösung gibt es verschiedene Tools und Befehle, mit denen virtuelle Maschinen verwaltet werden können:

  • virt-manager: Eine grafische Benutzeroberfläche zur Verwaltung von VMs unter KVM.
  • virsh: Ein Kommandozeilenwerkzeug zur Verwaltung von VMs, das mit KVM verwendet wird. Es ermöglicht das Erstellen, Starten und Stoppen von VMs.
  • docker ps: Zeigt alle laufenden Docker-Container an.
  • podman ps: Zeigt alle laufenden Podman-Container an.
