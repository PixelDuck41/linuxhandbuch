5. Netzwerkverwaltung

Zurück zum Start

Die Netzwerkverwaltung in Linux umfasst die Konfiguration von Netzwerkschnittstellen, das Zuweisen von IP-Adressen, die Verwaltung von Gateways und DNS-Servern sowie die Durchführung von Netzwerkfehlerbehebungen. In diesem Abschnitt werden die grundlegenden Befehle und Techniken zur Verwaltung und Diagnose von Netzwerken behandelt.

Netzwerkkonfiguration

Linux bietet eine Vielzahl von Tools zur Netzwerkkonfiguration. Die gängigsten Tools zur Verwaltung von Netzwerkschnittstellen sind ifconfig und ip.

Verwendung von ifconfig

Das Kommando ifconfig wird verwendet, um Netzwerkschnittstellen zu konfigurieren und die Netzwerkverbindungen zu überwachen.

  • ifconfig: Zeigt die Konfiguration der Netzwerkinterfaces an.
    Beispiel: ifconfig - Zeigt alle aktiven Netzwerkinterfaces an.
  • ifconfig eth0: Zeigt die Konfiguration des Interfaces eth0 an.
    Beispiel: ifconfig eth0 - Zeigt detaillierte Informationen über das Interface eth0 an.
  • ifconfig eth0 up: Aktiviert das Netzwerkinterface eth0.
    Beispiel: sudo ifconfig eth0 up - Aktiviert das Netzwerkinterface.
  • ifconfig eth0 down: Deaktiviert das Netzwerkinterface eth0.
    Beispiel: sudo ifconfig eth0 down - Deaktiviert das Netzwerkinterface.

Verwendung von ip

Das ip-Kommando ist das modernere und bevorzugte Werkzeug zur Netzwerkadministration. Es ersetzt ifconfig und bietet eine mächtigere Schnittstelle zur Konfiguration von Netzwerkschnittstellen.

  • ip a: Zeigt alle Netzwerkinterfaces und deren IP-Adressen an.
    Beispiel: ip a - Listet alle Netzwerkschnittstellen mit deren aktuellen IP-Adressen auf.
  • ip addr add 192.168.1.100/24 dev eth0: Fügt eine IP-Adresse zu einem Interface hinzu.
    Beispiel: sudo ip addr add 192.168.1.100/24 dev eth0 - Fügt die IP-Adresse `192.168.1.100` zum Interface eth0 hinzu.
  • ip link set eth0 up: Aktiviert das Interface.
    Beispiel: sudo ip link set eth0 up - Aktiviert das Interface eth0.
  • ip link set eth0 down: Deaktiviert das Interface.
    Beispiel: sudo ip link set eth0 down - Deaktiviert das Interface eth0.

IP-Adressen, Subnetze, Gateways

Ein wichtiger Teil der Netzwerkkonfiguration ist das Zuweisen von IP-Adressen und das Definieren von Subnetzen und Gateways.

  • IP-Adresse: Eine IP-Adresse wird verwendet, um ein Gerät im Netzwerk eindeutig zu identifizieren. Beispiel: `192.168.1.100`.
  • Subnetzmaske: Eine Subnetzmaske definiert, welcher Teil einer IP-Adresse das Netzwerk und welcher Teil das Gerät im Netzwerk darstellt. Beispiel: `255.255.255.0`.
  • Gateway: Ein Gateway ist ein Gerät, das als Vermittler zwischen verschiedenen Netzwerken dient. In der Regel handelt es sich dabei um den Router.

Beispiel: IP-Adresse und Gateway setzen mit ip

  • ip addr add: Zuweisen einer IP-Adresse.
    Beispiel: sudo ip addr add 192.168.1.100/24 dev eth0 - Setzt die IP-Adresse `192.168.1.100` für das Interface eth0.
  • ip route add default via: Setzt das Standard-Gateway.
    Beispiel: sudo ip route add default via 192.168.1.1 - Setzt das Standard-Gateway auf `192.168.1.1`.

Netzwerkanalyse und Fehlerbehebung

Die Netzwerkanalyse und Fehlerbehebung sind ein wichtiger Teil der Netzwerkverwaltung. Hier sind einige der gängigsten Tools und Befehle, die dir helfen können, Netzwerkprobleme zu diagnostizieren.

  • ping: Mit diesem Befehl kannst du testen, ob eine Verbindung zu einem anderen Host im Netzwerk besteht.
    Beispiel: ping 192.168.1.1 - Sendet Echo-Anfragen an die IP-Adresse `192.168.1.1`.
  • traceroute: Zeigt die Route, die ein Paket zu einem Zielhost nimmt, und hilft dabei, Netzwerkengpässe zu identifizieren.
    Beispiel: traceroute www.example.com - Zeigt den Pfad zum Host `www.example.com`.
  • netstat: Zeigt Netzwerkverbindungen, Routing-Tabellen und Netzwerkstatistiken an.
    Beispiel: netstat -tuln - Listet alle offenen Ports und Verbindungen auf.
  • ss: Ein modernerer Ersatz für netstat, der eine detailliertere Ansicht über Sockets und Verbindungen bietet.
    Beispiel: ss -tuln - Zeigt ebenfalls alle offenen Ports und Verbindungen an.

Firewall und Sicherheit

Die Konfiguration der Firewall ist ein entscheidender Bestandteil der Netzwerksicherheit. Linux bietet verschiedene Tools wie iptables, ufw und firewalld, um die Netzwerksicherheit zu gewährleisten.

Verwendung von ufw (Uncomplicated Firewall)

ufw ist eine benutzerfreundliche Front-End für die Verwaltung von iptables. Mit ufw kannst du einfach Firewall-Regeln hinzufügen und entfernen.

  • ufw enable: Aktiviert die Firewall.
    Beispiel: sudo ufw enable - Aktiviert ufw.
  • ufw allow: Erlaubt den Zugriff auf einen bestimmten Port oder Dienst.
    Beispiel: sudo ufw allow 80 - Erlaubt den HTTP-Zugriff (Port 80).
  • ufw status: Zeigt den aktuellen Status der Firewall und alle aktiven Regeln an.
    Beispiel: sudo ufw status - Zeigt alle aktiven Regeln an.
